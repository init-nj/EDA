## What is this project?

An analysis of Vrinda Store's e-commerce sales data to uncover patterns in customer behaviour, product performance, and regional trends — with the goal of supporting better business decisions.

## Dataset

- **31,047 orders × 21 columns**
- Covers customer demographics, order details, product categories, shipping info, and sales channels
- No missing values, no duplicates — clean and analysis-ready

## What's inside the notebook?

1. **Know Your Data** — loading, shape, nulls, duplicates
2. **Understanding Variables** — column descriptions, statistical summary
3. **Data Wrangling** — type fixes, standardization, feature engineering
4. **Visualizations** — 12 charts following the UBM rule (Univariate → Bivariate → Multivariate)
5. **Business Recommendations** — actionable insights for the store
6. **Conclusion**

## Key Findings

- 🏆 **Sets** are the top-selling category; **Bottoms** are the weakest
- 📦 **~92% of orders are delivered** successfully; 6.1% are returned or cancelled
- 🛒 **Amazon, Myntra & Flipkart** drive the majority of sales
- 📍 **Maharashtra, Karnataka & Uttar Pradesh** are the top revenue states
- 📅 **Friday** is the peak sales day; **Monday** is the slowest
- 👤 Core customers are **working-age adults (28–47)** making single-item purchases

## How to run

1. Clone the repo
2. Place `Vrinda_Store.csv` in the same folder as the notebook
3. Install dependencies: `pip install pandas numpy matplotlib seaborn plotly`
4. Run all cells top to bottom — the notebook is designed to execute in one go without errors
